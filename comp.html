DIFFERENCES BETWEEN COMPARISON OPERATOR AND LOGICAL OPERATORS

COMPARISON OPERATORS

Comparison operators are used to evaluate and compare values in programming. Here's everything you need to know:

TYPES OF COMPARISON OPERATORS

  1. Equal To(==): Checks if two values are equal.
    Example: 5=5 returns true
  2. Not Equal To(!=): Checks if two values are not equal.
    Example: 5 !=3 returns true
  3. Strictly Equal To(===): Checks if two values are equal and of the same data type.
    Example: 5 === "5" returns false
  4. Strictly Not Equal To(!==): Checks if two values are not equal to or not of the same data type.
    Example: 5 !== "5" returns true
  5. Greater Than(>): Checks if one value is greater than the other.
    Example: 5 > 3 returns true
  6. Less Than(<): Checks if one value is less than another.
    Example: 5 < 3 returns false
  7. Greater Than or Equal To(>=): Checks if one value is greater than or equal to another.
    Example: 5 >= 5 returns true
  8. Less Than or Equal To(<=): Checks if one value is less than or equal to one another.
    Example: 5 <= 3 returns false

Key Characteristics

  1. Return Value: Comparison operators return a boolean value (true or false).
  2. Data Type: Can compare numbers, strings, booleans, and other data types.
  3. Precedance: Evaluated before logical operators.

LOGICAL OPERATORS

Logical operators are used to combine conditional statements in programmiong, enambling your code to make decision based on multiple conditions.

TYPES OF LOGICAL OPERATORS

  1. AND(&&): Returns true if both conditions are true.
    Example: true && true returns true
  2. OR(||): Returns true if at least one condition is true.
    Example: true || false returns true
  3. NOT(!): Reverses the truthiness of a condition.
    Example: !true returns false

Key Characteristics

  1. Return Value: Logical operators return a boolean value (true or false).
  2. Short-Circuiting: Logical operators evaluate conditions sequentially and stop evaluating as soon as the result is determined.
  3. Precedance: Evaluated after comparison operators.
